基础与配置
命令	作用
git init	在当前目录初始化一个新的Git仓库。
git clone <url>	克隆一个远程仓库到本地。
git config --global user.name "Your Name"	设置全局用户名。
git config --global user.email "you@example.com"	设置全局用户邮箱。
git config --list	查看当前所有Git配置。
git help <command>	显示某个命令的帮助文档。
🔄 日常操作与提交流程
命令	作用
git status	高频命令。查看工作区和暂存区的文件状态。
git add <file>	将指定文件添加到暂存区（Stage）。
git add .	添加当前目录所有新文件和修改到暂存区（不含删除）。
git add -u	添加所有已跟踪文件的修改和删除到暂存区（不含新文件）。
git add -A	相当于 git add . 和 git add -u 的合集，添加所有变更。
git commit -m "message"	将暂存区内容提交到本地仓库。
git commit -am "message"	跳过 git add，直接提交所有已跟踪文件的修改（不含新文件）。
git commit --amend -m "new message"	修改最近一次提交的信息或内容。
🌿 分支管理
命令	作用
git branch	列出所有本地分支。
git branch -a	列出所有本地和远程分支。
git branch <branch-name>	创建一个新分支。
git switch <branch-name>	推荐。切换到指定分支。
git switch -c <branch-name>	推荐。创建并切换到新分支。
git checkout <branch-name>	切换到指定分支（旧命令，switch是其升级版）。
git checkout -b <branch-name>	创建并切换到新分支（旧命令）。
git merge <branch-name>	将指定分支合并到当前分支。
git merge --no-ff <branch-name>	禁用快进合并，保留分支历史信息。
git branch -d <branch-name>	删除已合并的分支。
git branch -D <branch-name>	强制删除未合并的分支（谨慎使用）。
🌍 远程仓库协作
命令	作用
git remote -v	查看所有远程仓库地址。
git remote add <name> <url>	添加一个新的远程仓库连接。
git fetch	从远程仓库获取最新数据，但不自动合并。
git fetch --prune	获取远程更新，并清理本地已删除的远程分支引用。
git pull	从远程拉取并合并到当前分支（fetch + merge）。
git pull --rebase	拉取并用变基方式合并，保持历史线性。
git push <remote> <branch>	将本地分支推送到远程仓库。
git push -u origin <branch>	首次推送分支并建立上游跟踪，之后可直接用 git push。
git push origin --delete <branch>	删除远程分支。
🔍 查看与对比
命令	作用
git diff	查看工作区与暂存区的差异。
git diff --staged / --cached	查看暂存区与上一次提交的差异。
git diff HEAD	查看工作区与最新提交的差异。
git log	查看提交历史记录。
git log --oneline	一行显示一条提交记录，更简洁。
git log --graph --oneline --all	以图形化方式查看所有分支的提交历史。
git show <commit-id>	查看某次提交的详细修改内容。
♻️ 撤销与回退
命令	作用	风险
git restore <file>	推荐。撤销工作区文件的修改。	低，丢弃的改动通常无法恢复
git restore --staged <file>	推荐。将文件从暂存区移回工作区（撤销add）。	低，不改变文件内容
git reset --soft <commit>	撤销到指定提交，保留工作区和暂存区的改动。	中，可找回，但会移动分支指针
git reset --mixed <commit>	默认。撤销到指定提交，保留工作区改动，清空暂存区。	中
git reset --hard <commit>	高危。彻底回退到指定提交，丢弃所有工作区和暂存区改动。	高，未提交的改动可能永久丢失
git revert <commit>	安全。创建一个新的提交来“反向”抵消指定提交的改动。	低，不改写历史，适合公共分支
📌 标签管理
命令	作用
git tag	列出所有标签。
git tag <tag-name>	创建一个轻量级标签（指向当前提交）。
git tag -a <tag-name> -m "message"	创建一个带备注的附注标签（推荐用于发布版本）。
git tag -d <tag-name>	删除本地标签。
git push origin <tag-name>	推送单个标签到远程仓库。
git push origin --tags	推送所有本地标签到远程仓库。
🧰 高级与实用技巧
命令	作用
git stash	临时保存当前工作区修改（“储藏”）。
git stash list	查看所有储藏记录。
git stash pop	应用最近的储藏并将其从储藏栈中删除。
git stash apply	应用最近的储藏，但保留在储藏栈中。
git cherry-pick <commit-id>	将指定提交的改动复制到当前分支。
git rebase <base-branch>	将当前分支的提交“变基”到目标分支上，使历史更线性。
git bisect	二分查找，用于定位引入Bug的提交。
git blame <file>	逐行查看文件是谁在什么时候修改的。
git reflog	查看本地仓库所有HEAD的历史操作记录，用于“后悔药”找回提交。
💡 使用技巧与建议
善用 git status：这是你的导航仪，任何操作前运行一下，它能清晰地告诉你当前所在分支、有哪些改动、以及下一步的建议。

用新命令替换旧命令：git switch 比 git checkout 职责更单一（专注于切换分支），git restore 也比 git checkout 更专注于撤销操作，建议优先使用它们。

提交前检查：在 git commit 前，最好先用 git diff 和 git diff --staged 确认一下即将提交的内容，避免误操作。
